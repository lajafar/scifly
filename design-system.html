Building a 1-0 Design System for Turbo mobile application

A messed up design process with chaos in design files and no documentation was a major issue for design and dev teams, resulting in a massive project backlog.

Team

Design systems designer (me), Design Team Lead, Mobile Developer

Deliverables

Informational Architecture, Design specs, Style guide, Component library

Key Performance Indicators

40%

Operational time reduction for the design team due to convenient and fast prototyping

70 20

Colors in Light mode palette, organized logically and hierarchically

ALL

Components in code have been unified without affecting the content hierarchy and visual identity of the product

Why are we doing this?

Optimize design workflow

Our design software couldn’t cope with the new challenges; no design artifacts or practices were documented in the files.
We had to reproduce many scenarios because the original screens were missed. Extra time was spent updating every iteration or mistake in Sketch, Zeplin and Basecamp.

Speed up development process

Chaos in design files was reflected in code as well. Development team often returned handoff materials due to mechanical mistakes and flaws in the screens, and lack of comments and guidance.

Focus on strategical and complex projects

The operational overload prevented the team from moving further and scaling the business. We risked losing competitive advantages in the market.

Highlights of my research

202 tickets* were delayed or backlogged due to design assets inaccuracy and chaos in code.

*over the 2021 year

Design assets were scattered amongst Google Workspace, Zeplin, Sketch and Basecamp.

It took twice as long to find references and up-to-date versions; lucky if there were any.

How might we reduce development delays, accelerate design-to-development and simplify the process of implementing changes?

Making it happen

Design assets restoration and migration to Figma

Figma had all the opportunities to store and organize design assets and artifacts with convenient access for the dev, design and product teams. I restored and migrated everything from Sketch and Zeplin to Figma, saving designers hours of searching.

Style guide

I identified patterns and organized colours, typography, grids and icons into a style kit. With the linked figma styles, we could quickly manage UI elements and finally had clean and easy-to-navigate files.

Component library and design documentation

Using the atomic design principle, I created components with variations and organized a library in Figma. Each component has a specific name and path in the library related to use cases (complex templates) or material design terminology (simple atoms and molecules). I also added use cases into the description of every component.

Design system handoff, launch and adoption through team training

Successfully launched all components and updated the code base. It was necessary for the design team to learn how to use the component library effectively, so I conducted a number of trainings to accelerate the adoption process.

Final result

Fast and precise "lego type" hi-fi prototype build-up using design system component library
